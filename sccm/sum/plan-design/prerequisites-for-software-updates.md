# <a name="prerequisites-for-software-updates-in-system-center-configuration-manager"></a>Необходимые условия для обновлений программного обеспечения в System Center Configuration Manager

*Применимо к: System Center Configuration Manager (Current Branch)*

В этом разделе перечислены необходимые условия для обновления программного обеспечения в System Center Configuration Manager. Для каждого из этих компонентов внешние и внутренние зависимости перечислены в отдельных таблицах.  

## <a name="software-update-dependencies-external-to-configuration-manager"></a>Внешние зависимости Configuration Manager для обновления программного обеспечения  
 В следующих разделах перечислены внешние зависимости для обновлений программного обеспечения.  

### <a name="internet-information-services-iis"></a>IIS  
 Службы IIS необходимы на серверах системы сайта, чтобы могла функционировать точка обновления ПО, точка управления и точка распространения. Дополнительные сведения см. в разделе [Необходимые компоненты для ролей системы сайта](../../core/plan-design/configs/site-and-site-system-prerequisites.md).  

### <a name="windows-server-update-services-wsus"></a>Службы Windows Server Update Services (WSUS)  
 Необходимы службы WSUS, позволяющие выполнять синхронизацию обновлений, а также сканирование клиентов для оценки соответствия требованиям обновления. Сервер служб WSUS должен быть установлен перед созданием роли системы сайта "Точка обновления программного обеспечения". Для точки обновления программного обеспечения поддерживаются следующие версии служб WSUS:  

-   WSUS 4 (роль в Windows Server 2012 и Windows Server 2012 R2)  

-   WSUS 3.2 (роль в Windows Server 2008 R2)  

 При наличии нескольких точек обновления программного обеспечения на сайте убедитесь, что они используют одинаковые версии WSUS.  

> [!WARNING]  
>  Поддержка классификации **обновлений** программного обеспечения была введена только начиная с WSUS 4.0. Прежде чем синхронизировать эту новую классификацию и получить возможность оценивать компьютеры Windows 10 в плане обслуживания Windows 10, важно установить [исправление 3095113](https://support.microsoft.com/kb/3095113) для служб WSUS на точках обновления программного обеспечения и серверах сайта. Это исправление позволяет WSUS на сервере под управлением Windows Server 2012 или Windows Server 2012 R2 синхронизировать и распространять обновления компонентов для Windows 10. Дополнительные сведения см. в разделе [Управление Windows как службой](../../osd/deploy-use/manage-windows-as-a-service.md).  
>   
>  В случае синхронизации обновлений программного обеспечения с классификацией **обновлений** до установки [исправление 3095113](https://support.microsoft.com/kb/3095113)ознакомьтесь с разделом [Recover from synchronizing the обновлений category before you install KB 3095113](#BKMK_RecoverUpgrades).  

### <a name="wsus-administration-console"></a>Консоль администрирования WSUS  
 Консоль администрирования WSUS необходима на сервере сайта Configuration Manager, если точка обновления программного обеспечения находится на удаленном сервере системы сайта, а службы WSUS еще не установлены на сервере сайта.  

> [!IMPORTANT]  
>  Версия WSUS на сервере сайта должна совпадать с версией WSUS на точках обновления программного обеспечения.  

> [!IMPORTANT]  
>  Не пользуйтесь консолью администрирования WSUS для настройки параметров служб WSUS. Диспетчер Configuration Manager подключается к службам WSUS в точке обновления ПО и настраивает соответствующие параметры.  

### <a name="windows-update-agent-wua"></a>Агент обновления Windows (WUA)  
 Клиент агента обновления Windows должен быть установлен на клиентах для подключения к серверу WSUS и получения списка обновлений, для которых необходимо проверить соответствие требованиям.  

 При установке Configuration Manager загружается последняя версия WUA. Затем, когда клиент Configuration Manager будет установлен, при необходимости производится обновление WUA. Если установка завершается неудачей, необходимо использовать другой метод обновления агента.  

## <a name="software-update-dependencies-internal-to-configuration-manager"></a>Внутренние зависимости Configuration Manager для обновления программного обеспечения  
 В следующих разделах перечислены внутренние зависимости для обновлений программного обеспечения в Configuration Manager.  

### <a name="management-points"></a>Точки управления  
 Точки управления передают данные между клиентскими компьютерами и сайтом Configuration Manager. Они необходимы для обновления программного обеспечения.  

### <a name="software-update-point"></a>Точка обновления программного обеспечения  
 Необходимо установить точку обновления программного обеспечения на сервере WSUS для развертывания обновлений ПО в Configuration Manager. Дополнительные сведения см. в разделе [Установка и настройка точки обновления программного обеспечения](../get-started/install-a-software-update-point.md).

### <a name="distribution-points"></a>точки распространения;  
 Точки распространения должны хранить контент для обновлений программного обеспечения. Дополнительные сведения об установке точек распространения и управлении содержимым см. в разделе [Управление содержимым и инфраструктурой содержимого](../../core/servers/deploy/configure/manage-content-and-content-infrastructure.md).  

### <a name="client-settings-for-software-updates"></a>Параметры клиентов для обновлений программного обеспечения  
 По умолчанию для клиентов включены обновления программного обеспечения. Однако существуют другие доступные параметры, которые определяют, как и когда клиенты оцениваются на соответствие для обновления программного обеспечения, а также управляют тем, как устанавливаются обновления программного обеспечения.  

 Дополнительные сведения см. в следующих разделах.  

-   [Параметры клиентов для обновлений программного обеспечения](../get-started/manage-settings-for-software-updates.md#a-namebkmkclientsettingsa-client-settings-for-software-updates)  

-   [Параметры клиентов для обновлений программного обеспечения](../../core/clients/deploy/about-client-settings.md#software-updates)  

### <a name="reporting-services-point"></a>Точка служб отчетов  
 Роль системы сайта точки служб отчетов может отображать отчеты об обновлении программного обеспечения. Эта роль является необязательной, но использовать ее рекомендуется. Дополнительные сведения о создании точки служб отчетов см. в разделе [Настройка отчетов](../../core/servers/manage/configuring-reporting.md).  

##  <a name="BKMK_RecoverUpgrades"></a> Восстановление после синхронизации категории обновлений до установки исправления 3095113  
 Необходимо установить [исправление 3095113](https://support.microsoft.com/kb/3095113) для служб WSUS на точки обновления программного обеспечения и серверы сайта, прежде чем синхронизировать классификацию **обновлений** . Если исправление не установлено, когда включена классификация **обновлений** , то службы WSUS обнаружат обновление компонентов сборки 1511 Windows 10, даже если не смогут корректно скачать и развернуть соответствующие пакеты. Если синхронизировать какие-либо обновления, не установив сначала [исправление 3095113](https://support.microsoft.com/kb/3095113), то вы заполните базу данных WSUS (SUSDB) непригодными для использования данными, которые придется очистить, чтобы корректно развертывать обновления.  Чтобы устранить эту проблему, используйте следующую процедуру.  

#### <a name="to-recover-from-synchronizing-the-upgrades-classification-before-you-install-kb-3095113"></a>Восстановление после синхронизации классификации обновлений до установки исправления 3095113  

1.  Удалите обновления программного обеспечения с помощью классификации обновлений. Можно использовать сценарий PowerShell, аналогичный следующему примеру сценария.  

    ```  
    $Server = Get-WSUSServer  
    $Config = $Server.GetConfiguration()  
    $Update10563 = “df4e45a3-946d-4467-b3fd-8621174bb666”  
    $UpdateGUID = New-Object Guid($Update10563)  
    $Server.DeleteUpdate($UpdateGUID)  
    ```  

    > [!IMPORTANT]  
    >  Скрипт необходимо запустить во всех точках обновления программного обеспечения в иерархии Configuration Manager, прежде чем переходить к следующему шагу.  

     Для массового удаления обновлений программного обеспечения с помощью классификации обновлений можно изменить сценарий PowerShell, чтобы считать набор идентификаторов GUID из TXT-файла.  

2.  Снимите флажок классификации **Обновления** в разделе "Свойства компонента точки обновления программного обеспечения" (см. раздел [Настройка классификаций и продуктов](../get-started/configure-classifications-and-products.md)), а затем запустите синхронизацию обновлений программного обеспечения (см. раздел [Синхронизация обновлений программного обеспечения](../get-started/synchronize-software-updates.md)).  

3.  Установите [исправление 3095113](https://support.microsoft.com/kb/3095113) для служб WSUS на точках обновления программного обеспечения и серверах сайта.  

4.  Установите флажок классификации **Обновления** в разделе "Свойства компонента точки обновления программного обеспечения" (см. раздел [Настройка классификаций и продуктов](../get-started/configure-classifications-and-products.md)), а затем запустите синхронизацию обновлений программного обеспечения (см. раздел [Синхронизация обновлений программного обеспечения](../get-started/synchronize-software-updates.md)).
