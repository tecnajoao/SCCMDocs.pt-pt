# <a name="manage-access-to-company-resource-based-on-device-network-and-application-risk"></a>Управление доступом к ресурсам организации на основе риска для устройства, сети и приложений

*Применимо к: System Center Configuration Manager (Current Branch)*

Вы можете управлять доступом с мобильных устройств к корпоративным ресурсам в соответствии с оценкой риска, осуществляемой Lookout, решением для защиты устройств от угроз, интегрируемым с Microsoft Intune. Риск оценивается на основе данных телеметрии, собираемых службой Lookout с устройств для получения сведения об уязвимостях операционной системы (ОС), установленных вредоносных программах и вредоносных сетевых профилях. 

На основе полученной из Lookout оценки риска, включенной в политиках соответствия требованиям System Center Configuration Manager (SCCM), можно настроить политики условного доступа и разрешить или заблокировать использование устройств, которые были определены как несоответствующие из-за обнаруженных на них угрозах.

[Гибридное развертывание системы управления мобильными устройствами (SCCM с Intune)](https://docs.microsoft.com/sccm/mdm/understand/choose-between-standalone-intune-and-hybrid-mobile-device-management) дает возможность контролировать доступ к корпоративным ресурсам и данным на основе оценки риска, предоставляемой такими решениями для защиты устройств от угроз, как Lookout.

## <a name="how-do-the-hybrid-mdm-deployment-and-lookout-device-threat-protection-help-protect-company-resources"></a>Как гибридное управление мобильными устройствами и служба защиты устройств от угроз Lookout помогают защитить корпоративные ресурсы?
Мобильное приложение Lookout (Lookout for Work), работающее на мобильных устройствах, собирает данные телеметрии по файловой системе, сетевому стеку, устройству и приложениям (если доступно), а затем отправляет их в облачную службу защиты устройств от угроз Lookout для вычисления совокупного риска для устройства, связанного с мобильными угрозами. Вы также можете изменить классификацию уровней риска в консоли Lookout в соответствии со своими требованиями.  

Политика соответствия требованиям в SCCM включает в себя новое правило для службы защиты устройств от угроз Lookout, которое основано на оценке риска угроз для устройства, получаемой из Lookout. Если это правило включено, устройство оценивается на соответствие требованиям.

Если устройство определяется как не отвечающее политике соответствия, доступ к таким ресурсам, как Exchange Online и SharePoint Online, может блокироваться с помощью политик условного доступа. Если доступ заблокирован, конечным пользователям предоставляется пошаговое руководство по устранению проблемы и восстановлению доступа к корпоративным ресурсам. Это руководство запускается через приложение Lookout for Work.

## <a name="supported-platforms"></a>Поддерживаемые платформы:
* **Android 4.1 и более поздних версий** с регистрацией в Microsoft Intune;
* **iOS 8 и более поздних версий** с регистрацией в Microsoft Intune.
Сведения о платформах и языках, поддерживаемых Lookout, см. в [этой статье](https://personal.support.lookout.com/hc/en-us/articles/114094140253).

## <a name="example-scenarios"></a>Примеры сценариев
Ниже приведены некоторые распространенные сценарии.
### <a name="control-access-based-on-threat-from-malicious-apps"></a>Управление доступом на основе угроз от вредоносных программ:
При обнаружении на устройстве вредоносных программ можно заблокировать на нем следующие возможности:
* подключение к корпоративной электронной почте до устранения угрозы;
* синхронизация корпоративных файлов с помощью приложения OneDrive для бизнеса;
* доступ к критически важным бизнес-приложениям.

**Блокирование доступа при обнаружении вредоносных программ:**

![схема блокирования доступа политикой условного доступа в случае, если устройство определяется как несоответствующее из-за наличия на нем вредоносных программ](media/config-mgr-maliciousapps_blocked.png)

**Устройство разблокировано и получает доступ к корпоративным ресурсам после устранения угрозы:**

![схема предоставления доступа политикой условного доступа после устранения угрозы и определения устройства как соответствующего](media/config-mgr-maliciousapps-unblocked.png)
### <a name="control-access-based-on-threat-to-network"></a>Управление доступом на основе угрозы для сети
Обнаруживайте угрозы для сети, такие как атаки типа "злоумышленник в середине", и ограничивайте доступ к сетям Wi-Fi на основе риска устройства.

**Доступ к сети через Wi-Fi заблокирован:**

![схема блокирования доступа через Wi-Fi политикой условного доступа в соответствии с угрозами для сети](media/config-mgr-network-wifi-blocked.png)

**Доступ восстановлен после исправления:**

![схема предоставления доступа политикой условного доступа после устранения угрозы](media/config-mgr-network-wifi-unblocked.png)
### <a name="control-access-to-sharepoint-online-based-on-threat-to-network"></a>Управление доступом к SharePoint Online в соответствии с угрозой для сети

Обнаруживайте угрозы для сети, такие как атаки типа "злоумышленник в середине", и запрещайте синхронизацию корпоративных файлов на основе риска устройства.

**Доступ к SharePoint Online заблокирован в соответствии с угрозой для сети, обнаруженной на устройстве:**

![Схема блокирования доступа к SharePoint Online политикой условного доступа из-за обнаружения угрозы](media/config-mgr-network-spo-blocked.png)


**Доступ восстановлен после исправления:**
