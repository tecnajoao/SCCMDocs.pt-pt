# <a name="pki-certificate-requirements-for-system-center-configuration-manager"></a>Requisitos de certificado PKI para o System Center Configuration Manager

*Aplica-se a: O System Center Configuration Manager (ramo atual)*

Os certificados de infraestrutura de chaves públicas (PKI) que poderá precisar para o System Center Configuration Manager estão listados nas tabelas seguintes. Estas informações pressupõem um conhecimento básico dos certificados PKI. Para obter orientações de implementação passo a passo, consulte [exemplo passo a passo de implementação de PKI certificados para o System Center Configuration Manager: Autoridade de certificação do Windows Server 2008](/sccm/core/plan-design/network/example-deployment-of-pki-certificates). 

Para obter mais informações sobre os serviços de certificados do Active Directory, consulte a seguinte documentação:  

-   Para o Windows Server 2012: [Descrição geral de serviços de certificados do Active Directory](http://go.microsoft.com/fwlink/p/?LinkId=286744)  

-   Para o Windows Server 2008: [Serviços de certificados do Active Directory no Windows Server 2008](http://go.microsoft.com/fwlink/p/?LinkId=115018)

Para obter informações sobre como utilizar a API de criptografia: Certificados de próxima geração (CNG) com o Configuration Manager, consulte [descrição geral de certificados CNG](cng-certificates-overview.md).


> [!IMPORTANT]  
> System Center Configuration Manager suporta certificados de proteger a 2 de algoritmo Hash (SHA-2). Certificados de SHA-2 trazem uma vantagem de segurança importantes. Por conseguinte, recomendamos o seguinte:
> - Emita novos certificados autenticação de servidor e cliente assinados com SHA-2, que inclui o SHA-256 e SHA-512, entre outros.
> - Todos os serviços de acesso à Internet devem utilizar um certificado de SHA-2. Por exemplo, se comprar um certificado público para utilização com um gateway de gestão na cloud, certifique-se de que adquirir um certificado de SHA-2.  
>
>Windows em vigor a partir de 14 de Fevereiro de 2017, já não confia determinados certificados assinados com SHA-1. Em geral, recomendamos que emita novo servidor e certificados de autenticação de cliente assinados com SHA-2 (que inclui o SHA-256 e SHA-512, entre outros). Além disso, recomendamos que quaisquer serviços de Internet para utilizam um certificado de SHA-2. Por exemplo, se comprar um certificado público para utilização com um gateway de gestão na cloud, certifique-se de que adquirir um certificado de SHA-2. "
>
> Na maioria dos casos, a alteração para SHA-2 certificados não tem qualquer impacto nas operações. Para obter mais informações, consulte [certificados do Windows a imposição de SHA1](http://social.technet.microsoft.com/wiki/contents/articles/32288.windows-enforcement-of-sha1-certificates.aspx).

 Com exceção dos certificados de cliente que o System Center Configuration Manager inscreve em dispositivos móveis e computadores Mac, certificados que o Microsoft Intune automaticamente cria para gerir dispositivos móveis e certificados de que o Centro de sistema O Configuration Manager instala em computadores baseados em AMT, poderá utilizar qualquer PKI para criar, implementar e gerir os seguintes certificados. No entanto, quando utilizar os serviços de certificados do Active Directory e modelos de certificado, esta solução PKI da Microsoft pode facilitar a gestão de certificados. Utilize a coluna **Modelo de certificado Microsoft a utilizar** nas tabelas seguintes para identificar o modelo de certificado que melhor corresponde aos requisitos de certificado. Apenas uma autoridade de certificação empresarial que executa no Enterprise Edition ou Datacenter Edition do sistema operativo de servidor, como o Windows Server 2008 Enterprise e Windows Server 2008 Datacenter, pode utilizar certificados com base no modelo.  

> [!IMPORTANT]
>  Quando utiliza uma empresa certificação autoridade e modelos de certificado, não utilize os modelos da versão 3. Estes modelos de certificado criam certificados que são incompatíveis com o System Center Configuration Manager. Em alternativa, utilize modelos da versão 2, utilizando as instruções seguintes:  
> 
> - Para uma AC no Windows Server 2012: Sobre o **compatibilidade** separador das propriedades do modelo de certificado, especifique **Windows Server 2003** para o **autoridade de certificação** opção, e  **Windows XP / Server 2003** para o **destinatário do certificado** opção.  
>   -   Para uma AC no Windows Server 2008: Ao duplicar um modelo de certificado, mantenha a seleção predefinida, **Windows Server 2003 Enterprise**, quando lhe for pedido o **modelo duplicado** caixa de diálogo. Não selecione **Windows Server 2008, Enterprise Edition**.  

 Utilize as secções seguintes para ver os requisitos de certificado.

Estas informações são fornecidas apenas para fins informativos.|Este certificado é automaticamente solicitado e instalado quando os utilizadores autenticados inscrevem os respetivos dispositivos móveis com o Microsoft Intune.
