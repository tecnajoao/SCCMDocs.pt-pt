*Aplica-se a: O System Center Configuration Manager (ramo atual)*

Este artigo disponibiliza as seções a seguir com vários recursos e dicas para usar a biblioteca de documentação do Configuration Manager:  

- [Como pesquisar](#bkmk_searchtips)  

- [Submeter erros de documento, aprimoramentos, perguntas e novas ideias](#bkmk_docfeedback)  

- [Como ser notificado de alterações](#bkmk_notifications)

Para obter ajuda geral com o produto, consulte [encontrar ajuda](/sccm/core/understand/find-help).


##  <a name="bkmk_searchtips"></a> Pesquisa   
 Utilize as seguintes sugestões de procura para localizar as informações de que necessita:  

-   Ao utilizar o motor de busca preferido para localizar os conteúdos para o Configuration Manager, incluir `SCCM` juntamente com as palavras-chave de pesquisa.  

    - Procure resultados a partir do docs.microsoft.com para o ramo atual do Configuration Manager. Resultados de technet.microsoft.com ou msdn.microsoft.com destinam-se a versões mais antigas do produto.  

    - Para além disso, concentre-se os resultados de pesquisa à biblioteca de conteúdos, inclua `site:docs.microsoft.com` para definir o âmbito o motor de busca.  

-   Utilize termos de pesquisa que correspondem a terminologia na interface do usuário e a documentação online. Evite os termos não oficiais ou as abreviaturas que poderá ver no conteúdo da Comunidade. Por exemplo, procure "ponto de gestão" em vez de "MP"; "tipo de implementação" em vez de "DT"; e "atualizações de software" em vez de "SUM".  

-   Para procurar um artigo que está sendo exibida, utilizar o browser **encontrar** funcionalidade. Com navegadores da web mais modernos, prima **Ctrl**+**F** e, em seguida, introduza os termos de pesquisa.  

-   Cada artigo no docs.microsoft.com inclui os campos seguintes para ajudar a pesquisar o conteúdo:  

    - **Pesquisa** no canto superior direito. Para pesquisar todos os artigos digite termos neste campo. Artigos na biblioteca do Configuration Manager incluem automaticamente o âmbito "ConfigMgr".  

    - **Filtrar por título** acima a tabela da esquerda do conteúdo. Para pesquisar o índice atual, introduza os termos neste campo. Este campo corresponde apenas termos que aparecem nos títulos de artigo para o nó atual. Por exemplo, infraestrutura de núcleo ou gestão de aplicações.  

- Está a ter problemas para encontrar alguma coisa? [Ficheiro comentários!](#bkmk_docfeedback) Quando submeter o problema, fornece o mecanismo de pesquisa que está a utilizar, as palavras-chave que tentou e o artigo de destino. Estes comentários ajuda a otimizar o conteúdo para pesquisa melhor de Microsoft.  



## <a name="bkmk_docfeedback"></a> Comentários

Vá para a secção de comentários na parte inferior, clicando a **comentários** ligação no canto superior direito da qualquer artigo. Esta secção é integrada com problemas do GitHub. Para obter mais informações sobre a integração com problemas do GitHub, consulte a [mensagem de blogue de plataforma do docs](https://docs.microsoft.com/teamblog/a-new-feedback-system-is-coming-to-docs).

Para partilhar comentários sobre o próprio produto do Configuration Manager, clique em **comentários sobre o produto**. Para obter mais informações, consulte [comentários sobre o produto](/sccm/core/understand/find-help#product-feedback). 

R [conta do GitHub](https://github.com/join) é um pré-requisito para fornecer comentários sobre a documentação. Depois de iniciar sessão, não há uma autorização de uso individual para MicrosoftDocs. Em seguida, quando clica em **comentários de conteúdos**, introduza um título e o comentário e, em seguida **submeter comentários**. Esta ação ficheiros de um novo problema para o artigo de destino na [SCCMdocs repositório](https://github.com/MicrosoftDocs/SCCMdocs/issues).

Esta integração também apresenta quaisquer problemas abertos ou fechados existentes para o artigo de destino. Se existir alguma, analisá-los antes de submeter um novo problema. Se encontrar um problema relacionado, clique no ícone de face para adicionar uma reação ou pode expandi-lo para adicionar um comentário. 

#### <a name="types-of-feedback"></a>Tipos de comentários
Utilize problemas do GitHub para submeter os seguintes tipos de comentários:
- Bug do documento: O conteúdo está desatualizada, claro, confuso ou interrompida.
- Aprimoramento do documento: Uma sugestão para melhorar o artigo.
- Pergunta de documento: Precisa de ajuda para localizar a documentação existente.
- Idéia de documento: Uma sugestão para um novo artigo. Utilize este método em vez do UserVoice para comentários sobre a documentação.
- Parabéns pelo artigo: Obter comentários positivos sobre um artigo útil ou informativo!
- Localização: Comentários sobre a conversão de conteúdo.
- Otimização do mecanismo de pesquisa (SEO): Comentários sobre problemas de pesquisa de conteúdo. Inclua o motor de busca, palavras-chave e o artigo de destino nos comentários.

Se problemas são gerados para os tópicos relacionados no documento, como [comentários sobre o produto](/sccm/core/understand/find-help#product-feedback), [perguntas de produto](https://social.technet.microsoft.com/Forums/en-US/home?category=ConfigMgrCB), ou [pedidos de suporte](https://aka.ms/cmcbsupport), estes problemas serão fechados e o utilizador redirecionado para o canal de comentários adequada.

Para partilhar comentários na plataforma docs.microsoft.com, veja [comentários a documentos](https://aka.ms/sitefeedback). A plataforma inclui todos os componentes de wrapper, como o cabeçalho, o sumário e o menu à direita. Também como os artigos de composição no navegador, como o tipo de letra, caixas de alerta e página âncoras.



## <a name="bkmk_notifications"></a> Notificações

Para receber notificações quando o conteúdo muda na biblioteca de documentação, utilize os seguintes passos:

1. Utilize o [docs pesquisa](https://docs.microsoft.com/search/index?scope=ConfigMgr) para encontrar um artigo ou um conjunto de artigos. Por exemplo:
    - Procurar um único artigo por título: ["Ficheiros de registo para resolução de problemas - Configuration Manager"](https://docs.microsoft.com/search/index?search=%22Log+files+for+troubleshooting+-+Configuration+Manager%22&scope=ConfigMgr)
    - Procurar qualquer artigo relativamente à [SQL](https://docs.microsoft.com/search/index?search=SQL&scope=ConfigMgr)
2. No canto superior direito, clique nas **RSS** ligação. 
3. Utilize este feed em qualquer aplicativo de RSS para receber notificações quando houver uma alteração a qualquer um dos resultados da pesquisa.


> [!Tip]  
> Também pode **Watch** a [SCCMdocs repositório](https://github.com/MicrosoftDocs/SCCMdocs) no GitHub. Esse método gera muita das notificações. Ele também não inclui alterações a partir de um repositório privado utilizado pela Microsoft.  



## <a name="bkmk_contribute"></a> Contribuir

Biblioteca de documentação do Configuration Manager, como a maioria dos conteúdos em docs.microsoft.com, é aberto no GitHub. Esta biblioteca aceita e incentiva contribuições da Comunidade. Para obter mais informações sobre como começar, consulte a [Guia do contribuinte](https://docs.microsoft.com/contribute). Criar uma [conta do GitHub](https://github.com/join) é o único pré-requisito.

#### <a name="basic-steps-to-contribute-to-sccmdocs"></a>Passos básicos para contribuir para SCCMdocs
1. O artigo de destino, clique em **editar**. Esta ação abre o ficheiro de origem no GitHub.  

2. Para editar o ficheiro de origem, clique no ícone de lápis.  

3. Faça as alterações na origem de markdown. Para obter mais informações, consulte [como utilizar Markdown para escrever artigos do Docs](https://docs.microsoft.com/contribute/how-to-write-use-markdown).  

4. Na secção de alteração de ficheiro de propor, introduza o comentário de consolidação pública que descreve *o que* é alterado. Em seguida, clique em **propor alteração do ficheiro**.  

5. Desloque para baixo e verifique se as alterações que efetuou. Clique em **criar pedido pull** para abrir o formulário. Descrever *por que motivo* efetuou esta alteração. Marcar o autor de artigo e pedir que ele Revise. Clique em **criar pedido pull**.  


### <a name="what-to-contribute"></a>O que contribuir

Se estiver interessado em contribuir, mas não sabe onde começar, consulte as seguintes sugestões:  

- Pesquise a lista de problemas para as etiquetas da Comunidade como destino:  
  - [good-first-issue](https://github.com/MicrosoftDocs/sccmdocs/issues?q=is:open+is:issue+label:good-first-issue)   
  - [help-wanted](https://github.com/MicrosoftDocs/sccmdocs/issues?q=is:open+is:issue+label:help-wanted)  

    Estas etiquetas são atribuídas pelos autores da Microsoft para problemas que são bons candidatos para contribuições da Comunidade.  

- Reveja um artigo para maior precisão. Em seguida, atualize o **MS** através de metadados `mm/dd/yyyy` formato. Esta contribuição ajuda a manter o conteúdo novo.  

- Adicione esclarecimentos, exemplos ou diretrizes com base na sua experiência. Esta contribuição utiliza o poder da Comunidade para compartilhar conhecimento.   

- Traduções corretas num idioma diferente do inglês. Esta contribuição aprimora a usabilidade de conteúdo localizado.  

> [!Note]  
> Contribuições grandes necessitam assinar um contrato de licença de contribuição (CLA), se não é um funcionário da Microsoft. GitHub automaticamente requer que inicie o presente contrato quando uma contribuição cumpre o limiar.  


### <a name="tips"></a>Sugestões

Ao contribuir para o docs do Configuration Manager, siga estas Diretrizes gerais:

- Não se Surpreenda-nos com pedidos pull de grandes dimensões. Em vez disso, [enviar um problema](https://docs.microsoft.com/sccm/core/understand/use-docs#bkmk_docfeedback) e iniciar uma discussão. Em seguida, pode acordamos numa direção antes de investir uma grande quantidade de tempo.  

- Leitura a [guia de estilo da Microsoft](https://aka.ms/MicrosoftStyle). Saber os [Top 10 dicas para a Microsoft estilo e voz](https://docs.microsoft.com/style-guide/top-10-tips-style-voice).  

- Utilize o [modelo de pedido pull](https://github.com/MicrosoftDocs/SCCMdocs/blob/master/PULL_REQUEST_TEMPLATE.md) como ponto de partida do seu trabalho.  

- Siga os [fluxo de trabalho do fluxo do GitHub](https://guides.github.com/introduction/flow/).  

- Blog e tweet (ou qualquer que seja) sobre as suas contribuições, com frequência!
