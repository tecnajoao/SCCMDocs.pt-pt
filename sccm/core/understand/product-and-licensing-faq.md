# <a name="frequently-asked-questions-for-configuration-manager-branches-and-licensing"></a>Perguntas mais frequentes para ramos do Configuration Manager e licenciamento

 *Aplica-se a: O System Center Configuration Manager (ramo atual), o System Center Configuration Manager (Long term Servicing Branch)*

## <a name="summary"></a>Resumo
Nestas perguntas frequentes aborda questões comuns de licenciamento sobre versões do System Center Configuration Manager Current Branch e term Servicing Branch (LTSB), disponíveis por meio de programas de licenciamento em Volume da Microsoft. Este artigo destina-se a fins informativos. Não substituir ou substituir alguma documentação sobre o licenciamento do System Center Configuration Manager. Para obter mais informações, consulte o licenciamento do produto para [System Center 2016](https://www.microsoft.com/en-us/licensing/product-licensing/system-center-2016.aspx) <!-- this link doesn't work without some language code --> e o [termos de produto](https://www.microsoftvolumelicensing.com/DocumentSearch.aspx?Mode=3&DocumentTypeId=53). Termos do produto descrevem os termos de utilização para todos os produtos da Microsoft no licenciamento por Volume.

Para obter mais informações sobre as funcionalidades do Configuration Manager, consulte a [página do produto](https://www.microsoft.com/cloud-platform/system-center-configuration-manager).




## <a name="product-and-licensing-faq"></a>Produtos e licenciamento FAQ

### <a name="bkmk_cb"></a> O que é o ramo atual?  
O ramo atual é a compilação pronto do System Center Configuration Manager que fornece um modelo de serviço Active Directory de produção. Este modelo de manutenção é como a experiência com o Windows 10 ou a opção de instalação do Windows Server 2016 Nano Server. Esta abordagem dá suporte a clientes que estão a mover à "cadência de cloud" e gostaria de inovar mais rapidamente. Com o ramo atual, modelo de serviço, os clientes do System Center Configuration Manager continuam a receber novos recursos e funcionalidades. Por esse motivo, apenas os clientes com Software Assurance ativo no licenças do System Center Configuration Manager ou com direitos de equivalente de subscrição, pode instalar e utilizar o atual ramo do System Center Configuration Manager.


### <a name="bkmk_ltsb"></a> O que é o longo termo Servicing Branch (LTSB)?  
Term Servicing Branch é uma compilação pronto do System Center Configuration Manager de produção. Destina-se para os clientes que permitem que o Software Assurance ou direitos de subscrição equivalente a expirar. Em comparação com o ramo atual, o LTSB tem [funcionalidades reduzidas](/sccm/core/understand/introduction-to-the-ltsb#features-that-are-not-available-in-the-ltsb-of-configuration-manager). Os clientes que permitem que o Software Assurance ou direitos de subscrição equivalente a expirar tem de desinstalar o atual ramo do System Center Configuration Manager. Os clientes que tenham direitos de licença perpétua para o System Center Configuration Manager, em seguida, podem instalar e usar a compilação LTSB da versão do System Center Configuration Manager que está atualizado no momento da expiração.


### <a name="bkmk_licensing-acronyms"></a> Já vi SA e L & SA utilizado no conteúdo de licenciamento. O que esses acrônimos dizer em relação ao System Center Configuration Manager?    
SA (Software Assurance) e L & SA (licença e Software Assurance) são opções de licença que concedem direitos de usar o System Center Configuration Manager. SA é uma opção para um cliente que está a renovar cobertura SA a partir de um contrato de anterior. L & SA são uma opção para um cliente comprar uma licença nova e a cobertura SA.
  - **Software Assurance (SA)**: Os clientes têm de ter SA ativo em licenças do System Center Configuration Manager ou direitos equivalentes de subscrição, para poder instalar e utilizar a opção de ramo atual do System Center Configuration Manager.    

    - Apesar de SA é opcional para alguns produtos da Microsoft, a única forma de obter direitos para utilizar o System Center Configuration Manager Current Branch é com SA *(ou direitos de subscrição equivalente)*. Para obter mais informações, consulte a [FAQ do Software Assurance](https://www.microsoft.com/en-us/licensing/licensing-programs/FAQ-Software-Assurance.aspx).<!--this link doesn't work without some language code-->

  - **Licenciamento da Microsoft e o Software Assurance (L & SA)**: Os clientes a compra de licenças novas para o System Center Configuration Manager tem de adquirir L & SA (a licença e cobertura SA).   

    - O SA concede direitos para utilizar o ramo atual.

    - Se sua SA expira, e ainda terá uma licença do System Center Configuration Manager, pode deixar de utilizar o ramo atual. Para obter mais informações, consulte as FAQ [se meu SA expira e tive L & SA, o que posso obter?](#bkmk_sa-expires)

Para obter mais informações sobre as ofertas de licença, veja [formas de comprar](https://www.microsoft.com/en-us/licensing/licensing-programs) <!--this link doesn't work without some language code--> e [termos de licenciamento de produto](http://www.microsoftvolumelicensing.com/ProductResults.aspx?doc=Product%20Terms,OST&fid=64).  


### <a name="bkmk_equiv-sub"></a> Li o termo "subscrição equivalente", quais os programas que que o referenciam?   
Subscrições equivalentes consulte programas, como [Enterprise Mobility + Security](http://www.microsoftvolumelicensing.com/ProductResults.aspx?doc=Product%20Terms,OST&fid=51) (EMS) ou [do Microsoft 365 Enterprise](https://www.microsoft.com/microsoft-365/enterprise). Podem existir outras pessoas, mas esses programas são as mais comuns. Os termos de produto de licenciamento do Microsoft Volume se refere a esses programas como licenças de equivalente de licença de gestão.


### <a name="bkmk_ems-expires"></a> Tenho o Enterprise Mobility + Security e ele expirado, o que tem faço agora?  
EMS concede direitos para utilizar o System Center Configuration Manager (ramo atual e term Servicing Branch). Quando estes direitos expirarem, já não tem direitos para utilizar qualquer uma das filiais e tem de desinstalar.  


### <a name="bkmk_sa-expires"></a> Se meu SA expira e tive L & SA, o que posso fazer?   
Se sua SA expirar após 1 de Outubro de 2016, dependendo de qual programa adquiriu L & SA em, pode manter uma licença perpétua para utilizar o term Servicing Branch (LTSB). Se utilizar atualmente o ramo atual, tem de desinstalá-lo e, em seguida, instalar o LTSB. Não há suporte para migrar para ou converter a LTSB do Current Branch.

Se sua SA expirou antes de 1 de Outubro de 2016, e retida anteriormente a uma licença perpétua para o System Center Configuration Manager, em seguida, sua única opção para utilização em curso é instalar e utilizar o System Center 2012 R2 Configuration Manager e seus service packs disponíveis. Era necessário desinstalar o ramo atual, quando sua SA expirado e volte a instalar essa versão anterior do produto. Não há suporte para migrar para versões superiores ou inferiores a partir do System Center Configuration Manager Current Branch a versões anteriores do Configuration Manager.   

Se utilizar o System Center Endpoint Protection, deve desinstalá-lo se seu SA expirar. System Center Endpoint Protection oferece não *L (licença)* direitos e sem direitos perpétuos.<!--506238--> 


### <a name="bkmk_owncb"></a> "Possuo" o ramo atual?   
Não. Está licenciado para utilizar o ramo atual, enquanto tiver SA ativo. Por exemplo, via *L & SA*, quando *SA* expirar, em seguida, tiver apenas *L (licença)* direitos, que inclui direitos para utilizar o ramo atual. Se sua L fornece direitos perpétuos, pode utilizar o term Servicing Branch do System Center Configuration Manager (ou o System Center 2012 R2 Configuration Manager se sua SA expirou antes de 1 de Outubro de 2016) em vez do ramo atual.


### <a name="bkmk_standalone"></a> Posso comprar o System Center Configuration Manager autónomo sem que seja SA?      
Não. A única forma de obter direitos para utilizar o System Center Configuration Manager é adquirir uma licença com SA ou através de uma subscrição equivalente. Existem programas de desenvolvedor (como MSDN) em que o System Center Configuration Manager está disponível para fins de desenvolvimento e teste, mas não utilização de produção.


### <a name="bkmk_update-rights"></a> Posso ver as atualizações para o System Center Configuration Manager fornecida a partir de dentro do meu console, como a versão 1610. Tenho de direitos para instalá-lo?   
Se tiver o Active Directory *SA*, fizer. Se não tiver SA Active Directory, tem de desinstalar o ramo atual e, em seguida, pode instalar o LTSB do System Center Configuration Manager. O LTSB não recebe atualizações para versões incrementais do System Center Configuration Manager, mas a receber atualizações de segurança com base no ciclo de vida do suporte.


### <a name="bkmk_csp"></a> Que adquirir através de um fornecedor de soluções Cloud (CSP) do Microsoft 365 ou do EMS, é necessário ter direitos para utilizar o System Center Configuration Manager? 
Sim, tem direitos para utilizar o System Center Configuration Manager para gerir clientes cobertos pela licença de EMS. Primeiro baixe e instale o [software de avaliação](https://www.microsoft.com/evalcenter/evaluate-system-center-configuration-manager-and-endpoint-protection). Em seguida, entre em contato com Support da Microsoft para obter a chave de licença.<!--issue472-->  


### <a name="bkmk_expiration-date"></a> Minha data de fim de subscrição é o mesmo que uma data de expiração do SA?    
Se *SA* ou a sua subscrição está ativa, tem direitos de uso para System Center Configuration Manager Current Branch. Uma subscrição ativa é equivalente de ter o Active Directory *SA*, mas não perpétua *"L" (licença)*. Assim que a sua subscrição terminar, terá de desinstalar o ramo atual, e não tem direitos para utilizar o LTSB.  

  
### <a name="bkmk_sql"></a> Quais são os direitos de uso associados com a tecnologia SQL fornecida com o System Center Configuration Manager?    
Todos os produtos do System Center incluem a tecnologia do SQL Server. Termos de licenciamento da Microsoft para estes produtos permitem a utilização de cliente da tecnologia do SQL Server apenas aos componentes de suporte System Center. Licenças de acesso de cliente do SQL Server não são necessárias para que utilizam. 
 
Direitos de uso aprovado para os recursos do SQL com o System Center Configuration Manager incluem:
 - Função de base de dados do site
 - Windows Server Update Services (WSUS) para a função de ponto de atualização de software
 - SQL Server Reporting Services (SSRS) para a função de ponto de relatórios
 - Função de ponto de serviço de armazém de dados
 - Réplicas de base de dados para funções de ponto de gestão
 - SQL Server AlwaysOn 

A licença do SQL Server que está incluída com o System Center Configuration Manager suporta a cada instância do SQL Server que instalar para alojar uma base de dados para o Configuration Manager. No entanto, podem executar apenas bases de dados para o Configuration Manager na lista anterior em que o SQL Server quando utiliza esta licença. Se uma base de dados para qualquer Microsoft adicional ou produto de terceiros partilhar o SQL Server, tem de ter uma licença separada para essa instância do SQL Server.
